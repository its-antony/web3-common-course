# Design: Part06 收益、风险与结构性陷阱

## Context

Part 6 是课程的核心风险教育模块，需要在以下约束条件下完成：

**教育目标**：
- 帮助学员理解 Web3 收益的真实来源
- 建立识别风险和陷阱的能力
- 提供可操作的风险防范建议

**内容约束**：
- 遵循课程文档编写规范
- 遵循内容研发规范
- 不能过度恐吓导致学员放弃学习
- 不能轻描淡写导致学员低估风险

**受众特征**：
- 学生 / Web2 上班族，无 Web3 基础
- 对「如何赚钱」有天然兴趣
- 容易被高收益吸引

## Goals / Non-Goals

### Goals
1. 建立「收益必有来源」的基础认知框架
2. 提供可操作的风险识别 checklist
3. 通过真实案例增强风险感知
4. 帮助学员建立个人风险边界

### Non-Goals
- 不深入技术细节（如 MEV 的具体算法）
- 不提供投资建议或具体策略
- 不覆盖所有类型的风险（聚焦最常见的）

## Decisions

### 决策 1：章节顺序设计

**选择**：按「认知递进」顺序组织，而非「风险严重程度」顺序

**顺序逻辑**：
1. 6.1 收益来源 → 先建立「钱从哪里来」的认知框架
2. 6.2 高 APY → 具体分析最常见的「诱惑」
3. 6.3 庞氏模型 → 揭示最严重的结构性陷阱
4. 6.4 MEV → 介绍隐性成本（大部分人不知道）
5. 6.5 跨链桥 → 技术风险维度
6. 6.6 监管合规 → 外部环境风险
7. 6.7 风险位置 → 总结与行动建议

**理由**：先建立认知框架，再逐步深入具体风险，最后回归个人决策

### 决策 2：案例选择策略

**选择**：聚焦 3 个标志性案例 + 若干小案例

**核心案例**：
1. **Luna/UST（2022.5）** - 算法稳定币崩盘，损失 400 亿美元
   - 适用章节：6.3
   - 教学价值：展示「看似完美的机制设计」如何瞬间崩溃

2. **FTX（2022.11）** - 中心化交易所暴雷
   - 适用章节：6.3
   - 教学价值：展示「信任中心化机构」的风险

3. **典型土狗项目模式** - 匿名团队、高 APY、锁仓机制
   - 适用章节：6.2、6.3
   - 教学价值：识别最常见的骗局模式

**辅助案例**（按章节分布）：
- 6.4 MEV：具体三明治攻击交易示例
- 6.5 跨链桥：Ronin Bridge（6 亿美元）、Wormhole（3.2 亿美元）
- 6.6 监管：SEC 诉讼 Coinbase、中国 924 通知

**理由**：核心案例深入分析，辅助案例点到为止

### 决策 3：风险量化表达方式

**选择**：使用「影响范围 + 发生概率」的定性描述，而非精确数字

**表达示例**：
- 高风险：「历史上多次发生，单次损失可达本金 100%」
- 中风险：「偶尔发生，可能损失 10-50% 本金」
- 低风险：「罕见发生，损失通常可控」

**理由**：
- 精确概率难以计算且可能误导
- 定性描述更适合通识课学员理解

### 决策 4：图表可视化策略

**选择**：主要使用 Mermaid 图表 + Markdown 表格，减少外部截图依赖

**各章节可视化方案**：

| 章节 | 可视化类型 | 内容描述 |
|-----|-----------|---------|
| 6.1 | Mermaid flowchart | 四种收益来源的资金流向图 |
| 6.2 | Markdown 表格 | APY 风险等级对照表 |
| 6.3 | Mermaid sequence | 庞氏资金流转时序图 |
| 6.3 | Mermaid timeline | Luna 崩盘 72 小时时间线 |
| 6.4 | Mermaid sequence | 三明治攻击过程图 |
| 6.5 | Markdown 表格 | 跨链桥安全事件汇总 |
| 6.6 | Markdown 表格 | 各国监管政策对比 |
| 6.7 | Mermaid flowchart | 风险自查决策树 |

**理由**：
- Mermaid 可版本管理、易修改
- 减少对外部数据平台截图的依赖（避免时效性问题）

### 决策 5：学习资料来源策略

**优先级**：
1. 官方事后分析报告（如 Luna/FTX 清算报告）
2. 权威媒体深度报道（The Block、Decrypt）
3. 研究机构分析（Messari、Chainalysis）
4. Binance Academy 等教育平台基础内容

**语言处理**：
- 英文优质资料必须收录
- 尽量找到中文对应内容或翻译版
- 必要时提供核心内容的中文摘要

## Risks / Trade-offs

### 风险 1：案例时效性
- **风险**：Web3 变化快，案例可能过时
- **缓解**：选择具有「教科书」价值的经典案例（Luna/FTX），而非近期热点

### 风险 2：风险描述引发恐慌
- **风险**：过于强调风险导致学员完全放弃 Web3
- **缓解**：每个风险点后附带「如何防范」或「理性看待」的内容

### 风险 3：内容深度把控
- **风险**：MEV 等话题太技术化可能超出通识课范围
- **缓解**：只讲「是什么」和「对我有什么影响」，不讲底层技术细节

### 风险 4：监管信息过时
- **风险**：各国监管政策变化频繁
- **缓解**：标注内容更新时间，描述监管「趋势」而非具体条款
