## ADDED Requirements

### Requirement: Part04 Token 与经济系统课程内容

Part04 模块 SHALL 提供 Token 和 Web3 经济系统的核心认知，帮助学员理解 Token 的本质、稳定币的机制以及代币经济学基础。

#### Scenario: 完整的课程内容结构
- **WHEN** 学员访问 Part04 模块
- **THEN** 应能看到 5 个完整的课程章节
- **AND** 每个章节包含学习目标、核心内容、案例、关键概念速查、学习资料、学习任务、FAQ

---

### Requirement: 4.1 Token 的基本功能与作用

课程 SHALL 在 4.1 章节解释 Token 的定义、类型和核心功能。

#### Scenario: 理解 Token 的定义
- **WHEN** 学员完成 4.1 章节学习
- **THEN** 学员能用自己的话解释「什么是 Token」
- **AND** 学员能说出 Token 与「加密货币」的区别

#### Scenario: 理解 Token 的类型
- **WHEN** 学员完成 4.1 章节学习
- **THEN** 学员能列出 Token 的 4 种主要类型（支付型、功能型、治理型、证券型）
- **AND** 学员能举例说明每种类型的典型代表

#### Scenario: 理解 Token 的功能
- **WHEN** 学员完成 4.1 章节学习
- **THEN** 学员能说出 Token 的核心功能（价值存储、支付媒介、权益凭证、激励工具）

#### Scenario: 配套可视化
- **WHEN** 学员阅读 4.1 章节
- **THEN** 应能看到 Token 类型分类图

---

### Requirement: 4.2 Token 股票 积分的本质区别

课程 SHALL 在 4.2 章节清晰对比 Token、股票、积分的本质差异，帮助学员避免混淆。

#### Scenario: 理解三者的法律地位差异
- **WHEN** 学员完成 4.2 章节学习
- **THEN** 学员能解释 Token、股票、积分在法律地位上的差异
- **AND** 学员能简述 Howey Test 的基本概念

#### Scenario: 理解三者的价值来源差异
- **WHEN** 学员完成 4.2 章节学习
- **THEN** 学员能解释三者的价值分别来自什么
- **AND** 学员能理解为什么 Token 的归零风险更高

#### Scenario: 配套对比表
- **WHEN** 学员阅读 4.2 章节
- **THEN** 应能看到 Token/股票/积分的对比表
- **AND** 对比维度应包含：发行方、法律地位、权利、价值来源、流动性、风险

---

### Requirement: 4.3 稳定币专题

课程 SHALL 在 4.3 章节深入讲解稳定币的机制、风险和典型案例。

#### Scenario: 理解稳定币的必要性
- **WHEN** 学员完成 4.3 章节学习
- **THEN** 学员能解释为什么需要稳定币
- **AND** 学员能说出稳定币的核心价值

#### Scenario: 理解三类稳定币机制
- **WHEN** 学员完成 4.3 章节学习
- **THEN** 学员能区分法币抵押型、加密资产抵押型、算法稳定币
- **AND** 学员能说出每类的代表项目（USDT、USDC、DAI、UST）

#### Scenario: 理解 USDT USDC DAI 的差异和风险
- **WHEN** 学员完成 4.3 章节学习
- **THEN** 学员能完成「USDT/USDC/DAI 对比表」学习任务
- **AND** 对比内容应包含：发行方、抵押机制、监管状态、核心风险

#### Scenario: 理解 UST Luna 崩盘案例
- **WHEN** 学员完成 4.3 章节学习
- **THEN** 学员能简述 UST 的锚定机制
- **AND** 学员能解释「死亡螺旋」是如何发生的
- **AND** 学员能总结从 UST 崩盘中学到的教训

#### Scenario: 配套可视化
- **WHEN** 学员阅读 4.3 章节
- **THEN** 应能看到稳定币分类图
- **AND** 应能看到 UST 死亡螺旋示意图

### Requirement: 4.4 通胀解锁与激励

课程 SHALL 在 4.4 章节介绍代币经济学（Tokenomics）的基础概念。

#### Scenario: 理解供应量概念
- **WHEN** 学员完成 4.4 章节学习
- **THEN** 学员能区分「总供应量」和「流通量」
- **AND** 学员能解释为什么这个区分很重要

#### Scenario: 理解解锁机制
- **WHEN** 学员完成 4.4 章节学习
- **THEN** 学员能解释什么是「解锁计划（Vesting）」
- **AND** 学员能说出大额解锁可能对价格造成的影响
- **AND** 学员知道如何查看项目的解锁计划

#### Scenario: 理解通胀通缩机制
- **WHEN** 学员完成 4.4 章节学习
- **THEN** 学员能区分通胀型和通缩型 Token
- **AND** 学员能举例说明销毁机制

#### Scenario: 理解激励机制
- **WHEN** 学员完成 4.4 章节学习
- **THEN** 学员能理解 Token 激励的作用
- **AND** 学员能区分「真实收益」和「Token 排放」

#### Scenario: 配套案例
- **WHEN** 学员阅读 4.4 章节
- **THEN** 应能看到大额解锁导致价格崩盘的真实案例
- **AND** 案例应包含数据支撑（解锁量、价格变化）

#### Scenario: 配套可视化
- **WHEN** 学员阅读 4.4 章节
- **THEN** 应能看到典型解锁计划时间线图
- **AND** 应能看到供应量与价格关系的概念图

### Requirement: 4.5 为什么大多数 Token 最终会失败

课程 SHALL 在 4.5 章节帮助学员理解 Token 失败的常见原因，建立风险意识。

#### Scenario: 了解 Token 失败率
- **WHEN** 学员完成 4.5 章节学习
- **THEN** 学员能了解大多数 Token 最终会失败的事实
- **AND** 学员能接受这是正常现象而非「全是骗局」

#### Scenario: 理解常见失败原因
- **WHEN** 学员完成 4.5 章节学习
- **THEN** 学员能列出至少 5 种 Token 失败的原因
- **AND** 学员能理解「庞氏结构」的识别方法

#### Scenario: 掌握红旗检查清单
- **WHEN** 学员完成 4.5 章节学习
- **THEN** 学员应掌握一份「Token 红旗检查清单」
- **AND** 清单应包含：团队背景、用途清晰度、收益合理性、代币分布、合约安全

#### Scenario: 建立健康心态
- **WHEN** 学员完成 4.5 章节学习
- **THEN** 学员应理解「不要 FOMO，不要 All-in」的重要性
- **AND** 学员应能在理解风险后做出自己的决定

### Requirement: 文档格式规范

Part04 所有文档 SHALL 遵循《课程文档编写规范》。

#### Scenario: 必要的文档模块
- **WHEN** 审核任意 Part04 章节文档
- **THEN** 文档应包含以下模块：
  - 学习目标（1-3 条，使用「能够……」句式）
  - 核心内容（使用三级标题组织）
  - 案例/故事（200-500 字）
  - 关键概念速查（表格形式，3-8 个概念）
  - 学习资料（必读 + 选读）
  - 学习任务（1-3 个可验证任务）
  - 常见问题 FAQ（2-5 个问题）

#### Scenario: 引用规范
- **WHEN** 审核 Part04 文档的学习资料
- **THEN** 必读资料应为 S/A 级来源
- **AND** 所有资料应标注来源和预计阅读时间

#### Scenario: 风险提示规范
- **WHEN** 审核 Part04 文档
- **THEN** 涉及投资风险的内容应有明确风险提示
- **AND** 不应包含任何投资建议或推荐具体 Token

---

### Requirement: 内容客观性

Part04 内容 SHALL 保持客观中立，避免偏见。

#### Scenario: UST Luna 案例的客观性
- **WHEN** 审核 4.3 章节的 UST/Luna 案例
- **THEN** 内容应客观陈述事实和机制
- **AND** 不应过度渲染情绪或使用煽动性语言
- **AND** 应明确这是特定案例，不代表所有算法稳定币

#### Scenario: Token 风险描述的平衡性
- **WHEN** 审核 4.5 章节
- **THEN** 内容应客观描述风险，不应一味否定
- **AND** 应帮助学员建立判断能力而非灌输恐惧
