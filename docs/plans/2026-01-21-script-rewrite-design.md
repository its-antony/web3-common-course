# Part 01 讲稿重写设计方案

## 背景

现有讲稿的问题：基本上是把 PPT 内容稍微扩充成句子，本质上就是在"念 PPT"，缺乏深度，没有发挥讲师录视频时应该有的"讲解"价值。

## 设计目标

重写 Part 01 讲稿作为样板，确认风格后可推广到其他 Part。

## 设计决策

| 维度 | 决策 |
|-----|------|
| 风格定位 | 专业讲师型：保持专业感，穿插故事、类比、提问，像大学老师讲课 |
| 扩展方式 | 混合模式：概念用类比、特征用案例、误解用反例、总结用提问 |
| 长度标准 | 标准版：每页 100-200 字，提供足够内容支撑但留有发挥空间 |
| 内容策略 | 深度整合 + 独立创作：文档有的好内容直接用，没有的自由发挥 |
| 结构方案 | 双层结构：每节开头写"讲解思路"，然后逐页详解 |

## 讲稿文件结构

```markdown
# Part 01：Web3 全局速览

## 讲解导读
（整个 Part 的 3-5 句话定位）

---

## 1.1 Web3 是什么，不是什么

### 本节讲解思路
（3-5 句话概括核心观点和串联逻辑）

### 第1页：标题页
（100-200 字讲解内容）

### 第2页：...
（100-200 字讲解内容）

---

## 1.2 从 Web1 到 Web3
（同样结构）

...
```

## 内容扩展策略

| 内容类型 | 扩展方式 | 示例 |
|---------|---------|------|
| 概念定义 | 类比 + 一句话总结 | "Web3 就像从租房到买房的转变" |
| 特征/原则 | 真实案例 | 讲"无需许可"时举 Uniswap 例子 |
| 常见误解 | 反例 + 正解 | "很多人以为...但实际上..." |
| 对比表格 | 挑重点讲 + 故事串联 | 不逐行念，用一个故事带出对比 |
| 小节总结 | 提问收尾 | "所以回到开头的问题..." |

## 各小节讲解思路设计

### 1.1 Web3 是什么，不是什么（约 8 页）

**讲解思路**：
> 这一节的目标是帮学员建立 Web3 的第一印象，核心是一句话："Web3 = 读 + 写 + 拥有"。我们先用互联网演进的故事引出这个定义，再通过"特朗普被封号"的案例让学员感受 Web2 和 Web3 的本质区别。最后澄清常见误解，避免学员带着错误认知进入后续学习。整节的情绪线是：好奇 → 理解 → 恍然大悟。

**各页扩展方向**：
- 定义页：用"租房 vs 买房"类比解释"拥有"的含义
- 核心特征页：每个特征配一个 15 秒能讲完的小例子
- 误解页：用"很多人第一次听说 Web3 时会想到..."引入，制造共鸣
- 案例页：深挖特朗普事件的细节，对比 Lens/Farcaster 的设计

### 1.2 从 Web1 到 Web3（约 9 页）

**讲解思路**：
> 这一节用时间线带学员走一遍互联网 30 年的演进。关键不是让学员记住年份，而是理解每一代互联网"解决了什么问题、又带来了什么新问题"。Web2 的核心矛盾（用户创造价值，平台拥有价值）是理解 Web3 存在意义的关键。讲完后学员应该能回答：为什么需要 Web3？

**各页扩展方向**：
- Web1 页：讲"那个年代上网是什么体验"，唤起记忆或想象
- Web2 页：用抖音/微信的例子，讲"你的粉丝真的是你的吗？"
- 核心矛盾页：这是重点，用具体数字说明平台从创作者身上赚了多少
- 对比表页：不逐行念，挑 2-3 个最有冲击力的对比深入讲

### 1.3 Web3 的定位（约 9 页）

**讲解思路**：
> 这一节回答"Web3 在更大图景中处于什么位置"。很多人误以为 Web3 是一个独立的平行世界，实际上它既是互联网的延伸，也是金融系统的补充。我们从技术栈分层帮学员建立全局视角，再用 TradFi vs DeFi 的对比让学员理解 Web3 金融的独特价值。跨境转账案例是本节的"啊哈时刻"——让学员真切感受到 Web3 解决了什么实际问题。

**各页扩展方向**：
- "不是平行世界"页：强调"你用的钱包 App 还是要从 App Store 下载"
- 技术栈分层页：用"盖房子"类比五层结构，不要让学员觉得太技术
- TradFi vs DeFi 页：不是说 DeFi 更好，而是讲各自适合什么场景
- 跨境转账案例：用具体数字（时间、费用）制造对比冲击

### 1.4 Web3 的周期性（约 10 页）

**讲解思路**：
> 这一节是"打预防针"——让学员在深入学习前就理解这个行业的波动性是常态而非异常。我们用 2013、2017、2021 三个历史周期讲故事，重点不是价格涨跌，而是"每轮周期留下了什么"。学员应该带走一个认知：周期会反复，但每一轮都在推动行业向前。这一节的情绪线是：震惊 → 理解 → 理性看待。

**各页扩展方向**：
- "过山车"页：用具体数字让学员感受波动幅度，但不要制造恐慌
- 历史周期页：每个周期讲一个标志性事件/人物，让数字有温度
- 驱动因素页：这是干货，帮学员建立分析框架而非只看价格
- 如何看待周期页：引用巴菲特或行业大佬的观点增加说服力

### 1.5 进入 Web3 的门槛与挑战（约 9 页）

**讲解思路**：
> 最后一节是"现实检查"——让学员对即将面临的挑战有心理准备。我们诚实地讲五大门槛：认知、技术、资金、信息、心理。重点是让学员知道"这些门槛是可以跨越的"，而不是劝退。本课程的价值定位也在这里点明：帮你降低认知门槛和信息门槛。结尾要给学员信心和继续学习的动力。

**各页扩展方向**：
- 各类门槛页：每个门槛配一个"真实踩坑故事"，制造共鸣
- 好消息页：强调门槛在降低，举钱包体验改善的例子
- 课程定位页：呼应开头，形成闭环
- 结尾页：用一个激励性的问题或观点收尾

## 讲稿样例

以下是 1.1 小节中"案例：从账号被封看差异"这一页的新旧对比：

### 旧讲稿

```markdown
## 第8页：案例：从「账号被封」看差异

来看一个真实案例。2021 年 1 月，特朗普的 Twitter 账号被永久封禁。
不管你对这件事持什么立场，它揭示了一个事实：在 Web2 世界，平台
拥有绝对的控制权。而在 Web3 的社交应用中，比如 Lens Protocol、
Farcaster，你的社交图谱存储在区块链上，没有任何单一平台可以
「封禁」你的社交身份。这就是 Web3 带来的变化。
```

### 新讲稿

```markdown
## 第8页：案例：从「账号被封」看差异

这个案例可能会让你重新思考「数字身份」这件事。

2021 年 1 月 8 日，Twitter 永久封禁了特朗普的账号。一夜之间，
他在这个平台上积累的 8800 万粉丝、发布的 5 万多条推文，全部
无法访问。我们先不讨论这个决定对不对——我想请你思考一个问题：
一个人花了十几年积累的数字影响力，真的应该可以被一家公司的
一个决定瞬间清零吗？

这就是 Web2 的本质：你以为粉丝是你的，其实是平台的；你以为
内容是你的，其实存在平台的服务器上。

Web3 的社交应用想改变这一点。在 Lens Protocol 或 Farcaster 上，
你的粉丝关系存储在区块链上。如果某个前端应用不让你用了，你可以
带着所有粉丝关系迁移到另一个应用——就像换一个浏览器访问同一个
网站。没有任何单一平台可以「封禁」你的社交身份。

当然，这也带来新问题：如果没有平台审核，谁来处理有害内容？
这些问题我们在后面的课程会深入讨论。
```

### 对比分析

| 维度 | 旧讲稿 | 新讲稿 |
|-----|-------|-------|
| 字数 | ~100 字 | ~280 字 |
| 具体数据 | 无 | 8800万粉丝、5万条推文 |
| 引发思考 | 直接给结论 | 先提问再解答 |
| 类比 | 无 | "像换浏览器访问同一个网站" |
| 延伸思考 | 无 | 点出新问题，埋下伏笔 |
