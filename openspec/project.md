# Project Context

## Purpose

**Web3 通识课 v2.0** - 面向学生和 Web2 上班族的 Web3 入门教育课程。

**核心目标**：
- 系统性建立完整 Web3 认知地图
- 帮助学员理解 Web3 是什么、钱从哪里来、风险在哪里
- 掌握基础实操技能，能独立完成链上交互
- 提供明确的参与路径与决策建议

## Tech Stack

- **内容格式**：Markdown (.md)
- **版本管理**：Git
- **文档规范**：遵循 `课程文档编写规范.md`

## Project Conventions

### 文件命名规范

**文件夹命名**：
- 格式：`Part{XX}-{简短主题}`
- 编号使用两位数字，从 00 开始
- 示例：`Part05-主流赛道与商业模式`

**文档命名**：
- 格式：`{大章节}.{小章节}-{章节标题}.md`
- 示例：`5.7-预测市场Polymarket.md`

**资源文件命名**：
- 图片：`{章节编号}-{描述}-{序号}.{格式}`
- 示例：`5.1-DeFi架构图-01.png`

### 文档结构规范

每篇文档必须包含以下模块：
1. **学习目标** - 使用「能够……」句式，1-3 条
2. **核心内容** - 使用三级标题组织知识点
3. **案例/故事** - 真实案例，200-500 字
4. **关键概念速查** - 表格形式，3-8 个概念
5. **学习资料** - 必读 + 选读，标注阅读时间
6. **学习任务** - 1-3 个可验证的任务
7. **常见问题 FAQ** - 2-5 个问题
8. **下一步** - 指向下一章节

### 中英混排原则

- **正文**：使用中文
- **专业术语**：首次出现用「中文（English）」格式，后续可直接用英文
- **项目/产品名**：保留英文原名（如 Polymarket、Uniswap、MetaMask）
- **缩写**：首次写全称，如「去中心化金融（Decentralized Finance, DeFi）」

### 写作风格

- 专业但不晦涩，像有经验的朋友在讲解
- 每段不超过 5 行
- 多用类比和表格对比
- 关键数据标注来源

### Git Workflow

- 每篇文档底部标注更新日期、编写人、审核人
- 格式：`最后更新：YYYY-MM-DD`

## Domain Context

**领域**：Web3 / 区块链 / 加密货币教育

**课程结构**（11 个 Part）：
- Part 0：导论
- Part 1：Web3 全局速览
- Part 2：区块链与加密系统
- Part 3：钱包、账户与资产安全
- Part 4：Token 与经济系统
- Part 5：主流赛道与商业模式（DeFi、NFT、GameFi、DAO、RWA、AI+Crypto 等）
- Part 6：收益、风险与陷阱
- Part 7：项目快速判断方法
- Part 8：行业角色与岗位
- Part 9：参与方式
- Part 10：路径建议与决策
- Part 11：资源导航

**目标学员**：
- 学生（对 Web3 感兴趣但无基础）
- Web2 上班族（想了解或转型到 Web3）

## Important Constraints

**内容约束**：
- ❌ 禁止承诺收益性表述（如「一定能赚钱」「财富自由」）
- ❌ 禁止过度营销或煽动性语言
- ❌ 禁止过于技术化的术语堆砌
- ✅ 必须客观呈现风险与陷阱
- ✅ 课程定位为「通识」，不承诺收益

**合规约束**：
- 涉及投资、交易内容需明确风险提示
- 遵守相关金融教育内容规范

## External Dependencies

1. 官方文档 / 白皮书（如比特币白皮书）
2. Binance Academy / Coinbase Learn

4. 优质 KOL 文章 / 视频
5. 研报

- 钱包：MetaMask、OKX Wallet
- 区块浏览器：Etherscan、Solscan
- 数据平台：DefiLlama、Dune、Nansen、Arkham、DeBank
- 信息源：Twitter/X、Discord、Telegram
