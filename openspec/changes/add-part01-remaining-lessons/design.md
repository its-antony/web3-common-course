# Design: Part01 剩余章节内容设计

## Context

Part01「Web3 全局速览」是课程的开篇核心章节，目标是帮助学员在最短时间内建立完整的 Web3 认知框架。1.1 节已完成，奠定了 Web3 定义和核心特征的基础。剩余 4 个章节需要在此基础上，逐步深化学员对 Web3 的全局理解。

## Goals / Non-Goals

### Goals
- 让学员理解 Web3 的历史演进脉络（1.2）
- 让学员明白 Web3 在更大生态中的位置（1.3）
- 让学员理解市场周期规律，建立理性预期（1.4）
- 让学员正视入门难度，做好心理准备（1.5）

### Non-Goals
- 不深入技术细节（留给 Part2）
- 不涉及具体操作（留给 Part3）
- 不讲具体项目投资（留给后续章节）

## Content Design

### 1.2 从 Web1 到 Web3：互联网的三次演变

**核心叙事线**：
```
信息 → 平台 → 价值
只读 → 读写 → 读写拥有
```

**知识点设计**：

| 时代 | 核心特征 | 代表产品 | 用户角色 | 商业模式 |
|-----|---------|---------|---------|---------|
| Web1 | 静态只读 | Yahoo、门户网站 | 消费者 | 广告/订阅 |
| Web2 | 交互平台 | 微信、抖音、Facebook | 创作者（但不拥有） | 平台抽成 |
| Web3 | 价值拥有 | 以太坊、Uniswap | 所有者 | 协议费/代币 |

**可视化**：
- Mermaid 时间线图：三代互联网演进
- 表格对比：各维度差异

**案例**：
- 创作者经济对比：YouTube vs Mirror（Web2 vs Web3 创作平台）

### 1.3 Web3 在互联网与金融体系中的位置

**核心叙事线**：
```
Web3 ≠ 独立世界
Web3 = 互联网的延伸 + 金融系统的另一种形态
```

**知识点设计**：

1. **技术视角**：Web3 在互联网技术栈中的位置
   - 基础设施层（区块链）
   - 协议层（DeFi、NFT协议）
   - 应用层（DApp）
   - 用户入口（钱包）

2. **金融视角**：Web3 与传统金融的关系
   - 不是取代，是补充
   - 开放金融 vs 许可金融
   - 链上资产 vs 链下资产

3. **定位总结**：
   - Web3 是互联网的「价值层」
   - Web3 是金融系统的「开放分支」

**可视化**：
- Mermaid 架构图：Web3 技术栈
- 对比图：Web3 与传统金融

### 1.4 为什么 Web3 周期性爆发又回落

**核心叙事线**：
```
周期 = 创新 + 投机 + 泡沫 + 出清 + 沉淀
周期不是坏事，是行业发展的必经之路
```

**知识点设计**：

1. **历史周期回顾**：
   - 2013 周期：比特币首次被大众关注
   - 2017 周期：ICO 狂潮与泡沫破裂
   - 2021 周期：DeFi Summer、NFT 热潮
   - 当前位置：周期中的哪个阶段

2. **周期驱动因素**：
   - 技术创新（底层突破）
   - 叙事更迭（新故事吸引资金）
   - 杠杆与投机（放大波动）
   - 监管与黑天鹅（加速出清）

3. **如何看待周期**：
   - 泡沫中有价值沉淀
   - 每轮周期后基础设施都更完善
   - 长期视角 vs 短期波动

**可视化**：
- 市场周期示意图（可引用 BTC 历史价格走势）
- 周期各阶段特征表

**案例**：
- 2017 ICO 泡沫 → 留下了以太坊生态
- 2021 NFT 热潮 → 留下了数字艺术基础设施

### 1.5 普通人进入 Web3 面临的真实难度

**核心叙事线**：
```
Web3 门槛真实存在，但可以跨越
了解难度 ≠ 劝退，而是做好准备
```

**知识点设计**：

1. **知识门槛**：
   - 新概念密集（区块链、DeFi、NFT...）
   - 需要跨领域知识（技术+金融+经济）
   - 信息更新快，需持续学习

2. **操作门槛**：
   - 钱包操作不直观
   - Gas 费理解成本
   - 跨链转账风险
   - 没有「忘记密码」

3. **风险识别门槛**：
   - 诈骗和钓鱼无处不在
   - 项目真假难辨
   - 合约授权风险
   - 市场波动剧烈

4. **心理门槛**：
   - FOMO（错过恐惧）
   - FUD（恐惧不确定怀疑）
   - 暴富心态 vs 学习心态

5. **实际建议**：
   - 从学习开始，不急于投入资金
   - 小额试错，建立真实体感
   - 区分「学习成本」和「投资风险」
   - 给自己设定明确边界

**可视化**：
- 门槛分类图
- 入门路径建议流程图
