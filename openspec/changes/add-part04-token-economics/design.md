# Design: Part04 Token 与 Web3 的经济系统

## Context

Part04 是 Web3 通识课的经济模块，处于课程体系中的关键位置：
- **前置**: Part01-03 建立了 Web3 概念和区块链技术基础
- **本模块**: 从「技术」过渡到「经济」，理解 Token 的本质和价值
- **后续**: Part05 需要用到 Token 知识理解 DeFi、NFT 等应用

**目标学员特征**：
- 已完成 Part01-03，理解区块链基础概念
- 可能接触过稳定币（如用 USDT 入金）但不理解其机制
- 对「空气币」「割韭菜」有模糊认知但缺乏判断框架

**核心挑战**：
1. Token 概念容易与股票、积分混淆，需要清晰区分
2. 稳定币机制涉及金融知识，需要降维解释
3. 代币经济学专业性强，需要用通俗案例说明
4. 内容敏感性：涉及投资风险，需要客观中立

## Goals / Non-Goals

### Goals
- 让学员理解 Token 的本质和主要类型
- 让学员能区分 Token、股票、积分的核心差异
- 让学员掌握主流稳定币的机制和风险
- 让学员理解代币经济学的基础概念
- 让学员具备初步判断 Token 可持续性的能力

### Non-Goals
- 不教授如何投资或交易 Token
- 不推荐任何具体 Token
- 不深入讲解智能合约或 ERC 标准的技术细节
- 不涉及 Token 法律合规的专业分析

## Decisions

### D1: 内容组织顺序

**决定**: 采用「概念 → 对比 → 案例 → 原理 → 风险」的递进结构

| 章节 | 定位 | 学员收获 |
|-----|------|---------|
| 4.1 Token 基本功能 | 建立概念 | 知道 Token 是什么、有什么用 |
| 4.2 Token/股票/积分 | 澄清误区 | 理解三者本质区别，不再混淆 |
| 4.3 稳定币专题 | 深入案例 | 掌握最常用 Token 的机制和风险 |
| 4.4 通胀解锁激励 | 理解原理 | 具备代币经济学基础认知 |
| 4.5 Token 失败原因 | 建立判断 | 能初步评估 Token 的可持续性 |

**理由**:
- 先建立基础概念，再逐步深入
- 稳定币放在中间作为「桥梁」，既是最常接触的 Token，也是理解机制的入口
- 失败原因放最后，帮助学员建立风险意识

### D2: 稳定币专题的深度控制

**决定**: 聚焦「机制 + 风险」，不深入讲解套利和 DeFi 用途

**内容边界**:
- ✅ 三类稳定币的锚定机制
- ✅ 各类稳定币的核心风险
- ✅ UST/Luna 崩盘案例
- ❌ 稳定币套利策略
- ❌ 稳定币在 DeFi 中的收益玩法（Part05 再讲）

**理由**: 本模块目标是「理解」而非「使用」，DeFi 应用场景留到 Part05

### D3: UST/Luna 案例的处理方式

**决定**: 作为「警示案例」而非「猎奇故事」

**处理原则**:
1. 客观陈述事实和机制，不渲染情绪
2. 重点解释「为什么会崩」而非「崩得多惨」
3. 提炼可复用的教训，不仅仅是「算法稳定币危险」
4. 明确标注这是历史案例，当前市场已有变化

**案例结构**:
```
1. 背景：UST 是什么，为什么曾被追捧
2. 机制：算法稳定币如何工作
3. 崩盘：触发因素和死亡螺旋
4. 教训：我们能学到什么
```

### D4: 代币经济学的通俗化策略

**决定**: 用「工资发放」类比解释解锁机制

**类比框架**:
| 代币经济学概念 | 通俗类比 |
|--------------|---------|
| 总供应量 | 公司股份总数 |
| 流通量 | 市场上可交易的股份 |
| 解锁计划 | 员工期权分批发放 |
| 通胀机制 | 增发新股 |
| 销毁机制 | 股票回购 |
| 激励机制 | 绩效奖金 |

**理由**: 目标学员多为上班族，用职场概念更容易理解

### D5: 可视化内容规划

**决定**: 每节至少 1 个 Mermaid 图表

| 章节 | 图表类型 | 内容 |
|-----|---------|------|
| 4.1 | 分类图 | Token 类型树状图 |
| 4.2 | 对比表 | Token/股票/积分三栏对比 |
| 4.3 | 流程图 | 三类稳定币锚定机制 |
| 4.3 | 时序图 | UST 死亡螺旋过程 |
| 4.4 | 时间线 | 典型解锁计划示意 |

| 4.5 | 决策树 | Token 红旗检查清单 |

### D6: 学习资料选择标准

**决定**: 优先中文资料，但核心概念补充英文权威来源

| 来源类型 | 优先级 | 示例 |
|---------|-------|------|
| 官方文档 | S | Tether 储备报告、USDC 审计报告 |
| 教育平台 | A | Binance Academy 稳定币教程 |
| 中文媒体 | A | Foresight News UST 崩盘复盘 |
| 英文媒体 | A | The Block、Decrypt 深度报道 |
| 数据平台 | B | DefiLlama、Token Unlocks |

**理由**: 目标学员英文能力参差，优先中文便于学习，但核心知识需要权威来源背书

## Risks / Trade-offs

### R1: 稳定币信息时效性

**风险**: 稳定币市场变化快，信息可能过时
**缓解**:
- 使用截图时标注日期
- 重点讲解「机制」而非「数据」
- 在文档中提示学员查阅最新数据

### R2: UST/Luna 案例敏感性

**风险**: 可能被理解为「算法稳定币都不行」
**缓解**:
- 明确说明这是特定案例，不代表所有算法稳定币
- 客观分析 UST 的具体问题，而非一刀切

### R3: Token 投资建议的边界

**风险**: 内容可能被误读为投资建议
**缓解**:
- 在文档中明确声明「本课程不构成投资建议」
- 聚焦「理解」而非「操作」
- 风险警示清晰明确

### R4: 代币经济学简化可能丢失准确性

**风险**: 通俗化可能导致概念不够准确
**缓解**:
- 在类比后补充准确定义
- 在「进阶阅读」中提供专业资料
- FAQ 中处理常见误解

## Content Architecture

### 4.1 Token 的基本功能与作用

```
1. Token 是什么
   - 定义：区块链上的数字凭证
   - 与「币」的区别（严格说比特币不是 Token）

2. Token 的主要类型
   - 支付型 Token（如 BTC、ETH 作为支付手段）
   - 功能型 Token（Utility Token，如平台服务凭证）
   - 治理型 Token（如 UNI、AAVE 投票权）
   - 证券型 Token（Security Token，受监管）

3. Token 的核心功能
   - 价值存储
   - 支付媒介
   - 权益凭证
   - 激励工具

4. Token 的技术实现（简述）
   - ERC-20 等标准
   - 为什么 Token 可以自由发行
```

### 4.2 Token ≠ 股票 ≠ 积分

```
1. 三者的直观区别
   - 表格对比：发行方、监管、权利、流动性

2. 法律地位差异
   - 股票：有明确法律保护
   - Token：法律定位模糊
   - 积分：商家自行定义
   - Howey Test 简介

3. 价值来源差异
   - 股票：公司盈利和资产
   - Token：网络效用和共识
   - 积分：商家承诺

4. 风险差异
   - 归零可能性对比
   - 维权难度对比

5. 为什么这个区分很重要
   - 避免把 Token 当股票投资
   - 理解 Token 的真实风险
```

### 4.3 稳定币专题

```
1. 什么是稳定币，为什么需要
   - 加密货币波动性问题
   - 稳定币的核心价值

2. 三类稳定币机制
   2.1 法币抵押型
       - USDT：机制、争议、风险
       - USDC：机制、监管合规、风险
   2.2 加密资产抵押型
       - DAI：超额抵押机制、清算风险
   2.3 算法稳定币
       - 原理简介
       - 为什么难以持续

3. USDT / USDC / DAI 对比表
   - 发行方
   - 抵押机制
   - 监管状态
   - 核心风险
   - 市值和使用场景

4. 案例：UST/Luna 崩盘
   - UST 的机制（铸造/销毁套利）
   - 为什么曾被追捧（Anchor 20% APY）
   - 崩盘过程（死亡螺旋）
   - 教训总结

5. 稳定币使用建议
   - 不要把稳定币当「绝对稳定」
   - 分散风险
   - 关注储备透明度
```

### 4.4 通胀、解锁与激励

```
1. 代币供应量基础
   - 总供应量 vs 流通量
   - 为什么这个区分重要

2. 解锁计划（Vesting）
   - 什么是解锁
   - 为什么要锁仓
   - 典型解锁计划（悬崖期 + 线性释放）
   - 如何查看项目解锁计划

3. 大额解锁的影响
   - 案例：某项目解锁导致价格崩盘
   - 供给增加 vs 需求不变 → 价格下跌
   - 「解锁」不等于「一定卖」但市场会预期

4. 通胀与通缩机制
   - 通胀型 Token（持续增发）
   - 通缩型 Token（销毁机制）
   - 双重机制（如 ETH）

5. 激励机制设计
   - 用 Token 激励什么行为
   - 激励是否可持续
   - 「真实收益」vs「Token 排放」

6. 如何看待代币经济学
   - 关注长期可持续性
   - 警惕「高通胀高收益」陷阱
```

### 4.5 为什么大多数 Token 最终会失败

```
1. Token 失败率有多高
   - 数据：xx% 的 Token 归零
   - 存活曲线

2. 常见失败原因
   2.1 无实际用途（空气币）
   2.2 经济模型不可持续（庞氏结构）
   2.3 团队跑路（Rug Pull）
   2.4 技术失败或被黑
   2.5 监管打击
   2.6 竞争失败

3. 「庞氏结构」识别方法
   - 收益来源是新资金还是真实业务
   - 是否需要不断拉新
   - 高收益是否可解释

4. Token 红旗检查清单
   - 团队匿名或无法验证
   - 没有清晰用途
   - 承诺不合理高收益
   - 代币大量集中在少数地址
   - 合约未开源或未审计

5. 健康心态
   - 大多数 Token 会失败是正常的
   - 不要 FOMO，不要 All-in
   - 理解风险后再决定是否参与
```
