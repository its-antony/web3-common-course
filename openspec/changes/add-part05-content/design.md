# Design: Part05 主流赛道与商业模式内容研发

## Context

Part05 是 Web3 通识课中内容最丰富的章节，涵盖 10 个子主题，每个主题都是独立的赛道或商业模式。需要在保证内容深度的同时，确保学员能快速建立对各赛道的基本认知。

**挑战**：
- 内容覆盖面广，需要平衡深度与广度
- 行业变化快，部分内容（如 AI+Crypto）需要最新资料
- 各赛道关联性强，需要体现整体视角

## Goals / Non-Goals

**Goals**：
- 帮助学员建立各主流赛道的基础认知
- 讲清楚每个赛道的核心价值主张和商业模式
- 客观呈现风险与问题，避免营销倾向
- 提供可验证的学习任务

**Non-Goals**：
- 不追求每个赛道的深度技术细节
- 不推荐具体投资标的
- 不预测价格走势

## Decisions

### 1. 内容组织结构

**决策**：每篇文档采用统一的 7 模块结构

```
1. 学习目标（1-3 条）
2. 核心内容
   - 是什么（定义）
   - 解决什么问题
   - 怎么运作（核心机制）
   - 主要类型/分类
3. 案例/故事（真实案例，200-500 字）
4. 关键概念速查（表格，3-8 个概念）
5. 学习资料（必读 2-3，选读 2-5）
6. 学习任务（1-2 个可验证任务）
7. FAQ（2-5 个问题）
```

**理由**：统一结构便于学员形成学习习惯，提高内容可预期性。

### 2. 案例选择原则

**决策**：优先选择以下类型案例
1. **头部项目**：如 Uniswap、Aave（DeFi）、BAYC（NFT）
2. **有教育意义的失败案例**：如 Axie Infinity 经济崩塌、Luna/UST
3. **近期出圈案例**：如 Polymarket 2024 大选

**理由**：头部项目代表性强；失败案例有警示作用；出圈案例增加学员兴趣。

### 3. 资料来源优先级

**决策**：严格遵循《内容研发规范》的来源分级

```
S 级（优先）：官方文档、白皮书、顶级研究机构
A 级（推荐）：Binance Academy、Coinbase Learn、知名媒体
B 级（参考）：一般媒体、社区讨论
C 级（谨慎）：个人博客、未验证信息
```

**必读资料要求**：仅使用 S/A 级来源
**选读资料要求**：不低于 B 级

### 4. 可视化策略

**决策**：每篇文档至少包含以下可视化元素之一

| 文档 | 建议可视化 |
|------|-----------|
| 5.1 DeFi | 协议分层架构图（Mermaid graph） |
| 5.2 NFT | NFT 应用场景分类（表格） |
| 5.3 GameFi | 经济循环流程图（Mermaid flowchart） |
| 5.4 SocialFi/DID | DID vs 传统账号对比表 |
| 5.5 RWA | RWA 代币化流程（Mermaid sequence） |
| 5.6 DAO | DAO 治理流程（Mermaid flowchart） |
| 5.7 预测市场 | 预测市场运作机制（Mermaid sequence） |
| 5.8 Meme 币 | Meme 币生命周期（表格） |
| 5.9 AI+Crypto | AI+Crypto 应用分类（表格） |
| 5.10 失败模式 | 常见失败模式总结（表格） |

### 5. 章节关联设计

**决策**：在各章节中适当引用其他章节内容，建立知识网络

```
5.1 DeFi → 引用 Part4 Token 概念、Part6 风险
5.2 NFT → 引用 Part4 Token 标准
5.3 GameFi → 引用 5.1 DeFi 机制、5.2 NFT 应用
5.4 SocialFi → 引用 5.2 NFT 身份应用
5.5 RWA → 引用 5.1 DeFi 借贷、Part4 稳定币
5.6 DAO → 引用 Part4 Token 治理
5.7 预测市场 → 引用 5.1 DeFi 机制
5.10 失败模式 → 综合引用前面所有案例
```

## Risks / Trade-offs

### 风险 1：内容过时
- **风险**：AI+Crypto、Meme 币等领域变化快
- **缓解**：
  - 标注内容时间戳
  - 侧重讲解持久性原理而非具体项目
  - 推荐学员关注实时信息源

### 风险 2：内容深度不足
- **风险**：10 个主题导致每个主题深度有限
- **缓解**：
  - 明确"通识课"定位，不追求专业深度
  - 提供进阶学习资料供有兴趣学员深入

### 风险 3：营销倾向
- **风险**：部分赛道（如 AI+Crypto）炒作成分高
- **缓解**：
  - 明确区分"真需求"与"炒作"
  - 客观呈现失败案例
  - 遵循课程约束：禁止承诺收益

## Content Research Strategy

### 每节课研发流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 搜索收集  →  2. 筛选评估  →  3. 提炼整合  →  4. 转化表达  →  5. 附注来源  │
└─────────────────────────────────────────────────────────────┘
```

### 搜索平台清单

| 类型 | 平台 | 适用场景 |
|-----|------|---------|
| 官方文档 | 项目官网、GitHub、白皮书 | 技术细节、机制原理 |
| 教育平台 | Binance Academy、Coinbase Learn | 入门概念、基础知识 |
| 中文媒体 | Foresight News、律动、深潮 | 中文解读、本土视角 |
| 英文媒体 | The Block、Decrypt、CoinDesk | 行业动态、深度分析 |
| 研究平台 | Messari、Delphi Digital | 研报、数据分析 |
| 数据平台 | DefiLlama、Dune、Token Terminal | 数据支撑 |

### 每节课资料收集目标

- 核心知识点：3-5 篇相关资料
- 每节课总计：8-15 篇资料
- 最终筛选：必读 2-3 篇 + 选读 2-5 篇

## Open Questions

1. **Meme 币章节尺度**：如何在不鼓励投机的前提下客观讲解 Meme 币现象？
   - 建议方案：侧重文化分析和风险警示，明确"不构成投资建议"

2. **AI+Crypto 时效性**：该领域变化极快，如何保证内容不过时？
   - 建议方案：侧重讲解底层逻辑（去中心化算力、数据市场的需求），减少具体项目推荐

3. **失败案例敏感性**：如何处理仍有争议的失败案例？
   - 建议方案：选择已有明确结论的案例，对争议案例呈现多方观点
