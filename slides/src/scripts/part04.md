# Part 04：Token 与经济系统

## 讲解导读

这一章我们要解决一个核心问题：Token 到底是什么？它跟你熟悉的股票、积分有什么本质区别？

很多人刚接触 Web3 时，会下意识地用旧概念来理解新事物。比如把 Token 当股票买，结果项目方跑路了发现无处维权。这一章就是要帮你建立正确的认知框架。

我们会从最基础的概念讲起，然后深入到稳定币这个特殊品类，再到代币经济学的核心机制，最后直面一个残酷的事实：为什么大多数 Token 会失败。理解这些，你才能在 Web3 世界里做出理性决策。

---

## 第1页：标题页

欢迎来到 Part 4，Token 与经济系统。

如果说前面几章是在教你认识 Web3 的基础设施，这一章就是在教你认识 Web3 的「货币体系」。Token 是整个 Web3 世界运转的血液，也是大多数人进入这个领域的第一个接触点。

但 Token 也是最容易被误解的概念。接下来我会帮你建立清晰的认知，让你真正理解 Token 是什么、不是什么。

---

## 4.1 Token 的基本功能与作用

### 本节讲解思路

这一节是基础中的基础。我们要回答一个看似简单的问题：Token 是什么？

我会先给你一个清晰的定义，然后用四大类型帮你建立分类框架，接着讲 Token 的核心功能，最后点出 Token 最重要的特性——可编程性。掌握这些基础概念，后面的内容才能真正听懂。

---

### 第2页：什么是 Token

让我先给你一个清晰的定义：**Token 是区块链上的数字资产凭证，代表某种价值、权利或用途**。

用一个类比来帮助理解：如果把区块链比作一个大型游乐园，那原生币比如 ETH 就是游乐园的「通用货币」，用来支付各种费用。而 Token 就像游乐园里各个设施发行的「专属票券」——有的能玩过山车，有的能换礼品。

从技术上说，Token 就是智能合约里的一条记录，记录着「谁拥有多少」。任何人都可以部署智能合约来发行 Token，这就是 Web3 的开放性。

---

### 第3页：Coin vs Token

这里有一个经常让新手困惑的问题：Coin 和 Token 有什么区别？

简单来说，**Coin 是区块链的原生资产**，比如 BTC、ETH、SOL，它们是各自区块链的「原住民」。而 **Token 是建立在现有区块链上的资产**，比如 USDT、UNI、SHIB，它们是「租户」，依附于某条公链存在。

但我要告诉你一个现实：**在日常交流中，这两个词经常混用**。你不必过于纠结，重要的是理解背后的逻辑，而不是咬文嚼字。

---

### 第4页：Token 的四大类型

Token 可以根据用途分为四大类。这个分类框架对你理解不同项目非常重要。

**第一类是支付型 Token**，用于支付和价值转移，典型代表是 USDT、USDC 这些稳定币。**第二类是功能型 Token**，用于访问某个产品或服务，比如 BNB 可以用来支付币安交易手续费，FIL 可以购买存储空间。**第三类是治理型 Token**，持有它就有投票权，可以参与项目决策，比如 UNI、AAVE。**第四类是证券型 Token**，代表真实资产的所有权，这类受监管较严格。

不过我要提醒你，很多 Token 同时具备多种属性。比如 ETH 既是支付手段，也被用于质押治理。**分类只是帮助理解的工具，不必死板套用**。

---

### 第5页：Token 的核心功能

理解了类型，再来看功能。Token 通常具备四个核心功能。

**价值存储**——持有 Token 作为资产保值，比如很多人持有 BTC 来对抗通胀。**支付媒介**——用 Token 进行交易支付，比如用 USDT 购买其他加密资产。**权益凭证**——Token 代表某种权利，持有 UNI 就拥有 Uniswap 的投票权。**激励工具**——用 Token 奖励用户行为，比如提供流动性可以获得 Token 奖励。

这四个功能你要记住，因为后面分析任何项目的 Token，都可以从这四个维度去理解。

---

### 第6页：Token 的可编程性

讲了这么多，Token 的真正价值到底在哪？答案是：**可编程性**。

传统资产有各种限制。发行股票要审批，跨境转账要走银行系统，交易有时间限制。但 Token 不一样——**无需许可即可发行**，任何人都可以创建；**全球即时流通**，7×24 小时不间断；**透明可追溯**，所有流转都在链上公开；**可组合性**，不同 Token 可以像乐高一样组合。

正是这种可编程性，让 Web3 能构建出传统金融难以实现的创新应用。这也是 DeFi 能存在的根本原因。

---

### 第7页：Token 技术标准

最后补充一点技术背景。在以太坊上，大多数 Token 遵循统一的技术标准。

最常见的是 **ERC-20**，它定义了 Token 的基本功能：转账、查询余额、授权等。还有 **ERC-721**，就是 NFT 标准，每个 Token 都是独一无二的。以及 **ERC-1155**，可以同时包含同质化和非同质化 Token。

你不需要理解技术细节，只需要知道一点：**正是因为有了统一标准，Token 才能在整个生态系统中自由流通**。这就像 USB 接口标准一样，让不同设备可以互相连接。

---

### 第8页：4.1 本节要点

这一节的核心要点，我帮你总结一下。

Token 是区块链上的数字资产凭证，Coin 和 Token 的区别在于是否是原生资产。Token 分为支付型、功能型、治理型、证券型四大类，但很多 Token 同时具备多种属性。Token 的核心功能包括价值存储、支付媒介、权益凭证和激励工具。

最重要的是理解 Token 的可编程性——这是 Web3 创新的基础。接下来我们要讲一个非常重要的话题：Token 跟股票、积分到底有什么区别。

---

## 4.2 Token ≠ 股票 ≠ 积分

### 本节讲解思路

这一节要纠正两个常见的误解。很多人把 Token 当股票买，以为持有就能分享收益；或者把 Token 当积分，以为只是兑换服务的工具。这两种理解都有问题。

我会先告诉你为什么这个区分很重要，然后从法律地位、所有权、风险等多个维度做对比，最后用 Howey Test 帮你理解监管逻辑。这一节的内容可能会让你有点不舒服，但直面现实才能保护自己。

---

### 第9页：常见误区

如果你第一次接触 Token，很可能会有这样的想法：「Token 就像公司股票，持有就能分享收益」，或者「Token 就像会员积分，用来兑换服务」。

这些类比在入门时有一定帮助，但如果你不清楚**本质区别**，可能会导致严重误判。最典型的情况是：把 Token 当股票投资，结果项目方跑路后发现——**无处维权**。

所以这一节，我要帮你把这三个概念彻底区分清楚。

---

### 第10页：Token vs 股票 vs 积分

让我用一张表帮你快速建立框架。

从发行方看，Token 由项目团队发行，股票由注册公司发行，积分由商家发行。从法律地位看，Token 多数没有明确定位，股票受证券法保护，积分由商家自定规则。从所有权看，Token 不代表公司股权，股票代表部分所有权，积分没有所有权含义。

最关键的是**归零风险**：Token 风险很高，股票有破产清算机制还能拿回一点，积分则是商家倒闭就失效。这个对比你要记住。

---

### 第11页：Token vs 股票：法律地位

让我重点讲讲 Token 和股票的区别，因为这是最容易混淆的。

股票受到严格的证券法监管，公司必须定期披露财务信息，投资者权益受法律保护。公司破产时，股东还能参与资产清算。

但 Token 呢？大多数国家尚无明确法律定位，项目方没有披露义务，投资者权益缺乏法律保护，项目失败后通常**血本无归**。

2022 年 FTX 交易所暴雷就是一个典型案例。用户资产被挪用，维权困难重重。这说明加密领域目前还缺乏传统金融那样的投资者保护机制。

---

### 第12页：持有 Token ≠ 持有股份

我必须非常直接地告诉你：**持有某个项目的 Token，不代表你拥有该项目的股权**。

你没有权利要求分红，你没有权利审查公司账目，你没有权利参与公司重大决策（除非是治理型 Token），项目方赚再多钱，也没有义务分给 Token 持有者。

打个比方：这就像你买了游乐园的游戏币，游乐园生意再好，也不会给你分红。游戏币只能在园内消费，不能让你成为股东。很多人亏钱就是因为没搞清楚这一点。

---

### 第13页：Howey Test：证券认定标准

这里要介绍一个重要概念：**Howey Test**。

美国 SEC 使用这个测试来判断某资产是否属于「证券」，有四个要素：金钱投资、投入共同事业、期待利润、利润来自他人努力。

**如果四个要素全部满足，就会被认定为证券**，项目方可能面临法律风险。这就是为什么很多项目强调自己的 Token 是「功能型」而非「投资品」。SEC 起诉 Ripple 的案子就涉及这个问题，官司打了好几年，说明 Token 的法律定性仍有巨大争议。

---

### 第14页：Token vs 积分

再来看 Token 和积分的区别。表面上都是「用来换东西的」，但有关键不同。

**存储位置不一样**：积分存在商家服务器上，商家可以随时修改规则甚至清零；Token 存在你自己的钱包里，你有完整控制权。

**可转让性不一样**：积分通常不能转让，绑定你的账户；Token 可以自由转让给任何地址，可以在交易所买卖。

**价值来源不一样**：积分的价值由商家承诺，商家可以单方面改变兑换规则；Token 的价值由市场供需决定，价格可能大幅波动。

---

### 第15页：两类常见错误

理解了这些区别，能帮你避免两类常见错误。

**第一类：把 Token 当股票，高估其价值**。「这个项目很有前景，买它的 Token 就等于投资这家公司」——错了。Token 价格可能与项目成功完全脱钩，项目方可能不断增发稀释你的持有，或者团队套现离场。

**第二类：低估归零风险**。「Token 跌了就拿着，总会涨回来的」——未必。股票背后有公司资产，清算时可能还能拿回一点；Token 背后通常没有「硬资产」，归零就是真的归零。**历史上大多数 Token 最终归零**，这是事实。

---

### 第16页：4.2 本节要点

这一节的核心内容，让我帮你提炼一下。

Token 持有者的权益远不如股东，不受证券法保护。Token 与积分的核心区别在于所有权和可转让性。不要把 Token 当股票投资，也不要低估归零风险。理解 Howey Test 有助于判断 Token 的法律风险。

这些内容可能让你有点沉重，但直面现实才能保护自己。接下来我们进入一个重要专题：稳定币。

---

## 4.3 稳定币专题

### 本节讲解思路

稳定币是普通人最常接触的加密资产，很多人进入 Web3 的第一步就是买 USDT。但稳定币并不像名字听起来那么「稳定」。

这一节我会从稳定币存在的意义讲起，然后介绍三种类型，重点对比 USDT、USDC、DAI，最后用 UST 崩盘这个惨痛案例来警示风险。理解稳定币，是在 Web3 里保护资产的基础。

---

### 第17页：什么是稳定币

**稳定币是价格锚定法定货币的加密货币**，通常锚定美元，目标是保持 1:1 的汇率。

为什么需要稳定币？因为加密货币波动性太大。BTC 一天涨跌 10% 很常见，如果用 ETH 买咖啡，早上能买 2 杯，晚上可能只能买 1 杯。这就没法作为日常支付手段。

稳定币解决了这个问题。行情不好时可以换成稳定币避险，日常交易用稳定币作为加密世界的「美元」，跨境转账可以 24 小时即时到账，DeFi 协议大多以稳定币计价。可以说，稳定币是 Web3 的基础货币。

---

### 第18页：稳定币的三种类型

稳定币按照抵押机制分为三类。

**第一类是法币抵押型**，发行方持有等额的美元储备，每发行 1 个稳定币，银行账户里就有 1 美元。代表是 USDT 和 USDC。优点是机制简单，缺点是中心化，依赖发行方信誉。

**第二类是加密资产抵押型**，用户存入 ETH 等加密资产作为抵押品，借出稳定币。因为加密资产波动大，通常需要超额抵押。代表是 DAI。优点是去中心化，缺点是资本效率低。

**第三类是算法稳定币**，不依赖抵押品，通过算法调节供需维持锚定。代表是已经崩盘的 UST。**目前没有成功的纯算法稳定币**，这一点要记住。

---

### 第19页：USDT vs USDC vs DAI

这三个是最主流的稳定币，让我做一个关键对比。

**USDT** 由 Tether 发行，市值最大，流动性最好，但储备透明度存疑，而且可以冻结你的地址。**USDC** 由美国公司 Circle 发行，月度审计最透明，合规性最好，但同样中心化，2023 年还因为硅谷银行事件脱锚过。**DAI** 由 MakerDAO 协议发行，高度去中心化，无法被冻结，但机制复杂，有抵押品清算风险。

**没有「最好」的稳定币，只有「最适合」的**。重视合规选 USDC，重视去中心化选 DAI，需要最大流动性选 USDT。最佳策略是分散持有。

---

### 第20页：稳定币的核心风险

稳定币有「稳定」两个字，但并不意味着没有风险。

**USDT 的核心风险**是储备构成不透明，外界很难验证 Tether 是否真的有那么多美元，加上监管风险。**USDC 的核心风险**是银行风险，2023 年硅谷银行倒闭时，Circle 有 33 亿美元存款在那里，USDC 一度跌到 0.87 美元。**DAI 的核心风险**是抵押品波动和清算风险，如果 ETH 暴跌，可能触发大规模清算。

所以我再强调一次：**没有绝对安全的稳定币**，分散持有是最佳策略。

---

### 第21页：稳定币脱锚历史

让我用几个历史事件来说明稳定币的风险。

2018 年 10 月，USDT 跌到 0.85 美元，市场恐慌 Tether 储备不足。2023 年 3 月，USDC 跌到 0.87 美元，因为硅谷银行倒闭。2020 年 3 月「黑色星期四」，DAI 也出现过脱锚，ETH 暴跌导致大规模清算。

这些事件说明，即使是法币抵押型稳定币，也不是「稳赚不赔」的。你需要了解风险，做好准备。

---

### 第22页：UST/Luna 崩盘警示

接下来我要讲一个惨痛的案例：UST 崩盘。

2022 年 5 月，曾经的第三大稳定币 UST 在几天内归零，连带 LUNA 从 80 多美元跌到不足 0.0001 美元。UST 市值从约 180 亿美元归零，LUNA 市值从约 400 亿美元归零，无数投资者血本无归，创始人 Do Kwon 被多国通缉。

这是加密史上最大的崩盘事件之一。为什么会发生？我来解释一下机制。

---

### 第23页：UST 的死亡螺旋

UST 是算法稳定币，依靠与 LUNA 的互相转换来维持锚定。理论上，当 UST 脱锚时，套利者可以销毁 UST 换取 LUNA，减少 UST 供给，让价格回升。

但现实是，当恐慌来临，这个机制变成了「死亡螺旋」：UST 大量抛售导致脱锚，套利者销毁 UST 铸造 LUNA，LUNA 供给暴增导致价格暴跌，需要更多 LUNA 才能换回 1 美元，LUNA 进一步稀释暴跌，信心彻底崩溃。

**关键问题是**：Anchor 协议提供约 20% 的 UST 存款年化收益，这个收益来源不可持续，完全靠项目方补贴。当补贴停止，一切就崩塌了。

---

### 第24页：UST 教训

从 UST 崩盘中，我们能学到什么？

**第一，「稳定」不等于「无风险」**。算法稳定币在极端情况下可能完全失效。**第二，高收益等于高风险**。20% 的「稳定」收益本身就是巨大的红旗。**第三，理解机制很重要**。如果你不理解收益从哪里来，你可能就是那个「收益来源」。**第四，分散风险**。不要把所有资金放在单一稳定币。

UST 的失败不代表所有算法稳定币都不行，但确实证明了纯算法机制的脆弱性。目前市场主流仍是法币抵押型和加密资产抵押型。

---

### 第25页：4.3 本节要点

这一节的核心内容，我来总结一下。

稳定币分为法币抵押型、加密资产抵押型、算法型三类。USDT 流动性最大，USDC 最合规，DAI 最去中心化，各有优劣。即使是法币抵押型稳定币也有脱锚风险，没有绝对安全的选择。

UST 崩盘证明了纯算法稳定币的脆弱性，高收益往往意味着高风险。接下来我们进入代币经济学的核心话题：通胀、解锁与激励。

---

## 4.4 通胀、解锁与激励

### 本节讲解思路

这一节是代币经济学的核心内容。很多人看到「年化 100%」就兴奋，但不知道这个收益从哪来，最后反而亏钱。

我会从最基础的供应量概念讲起，然后解释解锁机制和它对价格的影响，再讲通胀通缩的不同模式，最后教你区分「真实收益」和「Token 排放」。这些知识能帮你避免被高 APY 迷惑。

---

### 第26页：Token 供应量基础

在评估一个 Token 时，首先要理解它的供应量设计。这里有三个关键概念。

**总供应量**是 Token 的最大发行量上限，比如 BTC 永远只有 2100 万枚。**流通量**是当前市场上可自由交易的数量，比如 BTC 目前约 1950 万枚。**锁仓量**是被锁定暂时不能交易的数量，比如团队锁仓、投资人锁仓。

它们的关系是：流通量 + 锁仓量 ≤ 总供应量。理解这个关系，是后面所有内容的基础。

---

### 第27页：市值 vs FDV

接下来是两个重要指标：市值和 FDV。

**市值**的计算公式是价格乘以流通量。**FDV**（完全稀释市值）的计算公式是价格乘以总供应量。

举个例子，假设某 Token 价格 10 美元，流通量 1000 万枚，总供应量 1 亿枚。那市值是 1 亿美元，但 FDV 是 10 亿美元。

**这里有一个重要警示**：如果一个项目市值看起来「很低」，但 FDV 很高，说明还有大量 Token 未流通。未来解锁可能带来巨大抛压，价格可能大幅下跌。

---

### 第28页：解锁计划（Vesting）

**解锁是指 Token 分批释放给持有者的机制**。项目方通常会对团队成员、早期投资人、顾问、生态激励设置锁仓。

为什么要锁仓？对团队成员是防止拿钱就跑，对早期投资人是避免上线后立即砸盘，对顾问是绑定长期利益。这是一种利益绑定机制。

典型的解锁结构是：先有一段「悬崖期」完全锁仓，然后线性释放。比如团队 Token 可能是 12 个月悬崖期，之后 36 个月线性释放。

---

### 第29页：解锁相关术语

这里补充几个你会经常遇到的术语。

**悬崖期（Cliff）**是完全锁仓阶段，这期间一枚 Token 都不能动。**线性释放**是悬崖期后，每月或每周按比例释放。**TGE** 是 Token Generation Event，就是 Token 正式发行的时刻。**TGE 解锁比例**是发行时立即释放的比例，比如「10% TGE 解锁」意味着上线时就能卖 10%。

了解这些术语，你才能看懂项目的代币分配方案。

---

### 第30页：大额解锁的影响

为什么解锁会影响价格？这是经济学 101：**供给增加 + 需求不变 = 价格下跌**。

当大量 Token 解锁进入市场，持有者可能选择卖出套现，市场预期抛压会提前抛售避险，买盘不足以承接就会导致价格下跌。

但解锁不一定导致暴跌，关键看几点：解锁占流通量的比例是否超过 10%，解锁方是投资人还是团队还是生态，当前市场环境是牛市还是熊市。**解锁占流通量超过 10% 需要警惕**。

---

### 第31页：通胀型 vs 通缩型 Token

Token 按照供给变化可以分为通胀型和通缩型。

**通胀型 Token** 会持续增发新 Token，比如 DOT 年通胀率约 10%，用于质押奖励。通胀会稀释现有持有者的份额。**通缩型 Token** 通过销毁减少供给，比如 BNB 定期回购销毁。减少供给理论上有利于价格。

还有**双重机制**，比如 ETH 同时有质押奖励增发和手续费销毁。在网络活跃时，销毁量可能超过增发量，ETH 就变成通缩资产。

---

### 第32页：真实收益 vs Token 排放

这是理解 DeFi 收益的**关键区分**。

**真实收益**来自协议的实际业务收入，比如交易手续费分成、借贷利息收入。只要业务持续，收益就可持续。**Token 排放**来自增发新 Token，本质是稀释。总量有限，不可持续。

很多项目用高 Token 排放来吸引用户，标榜「年化 100%」，但如果 Token 价格下跌 80%，你实际上是亏钱的。**当你看到高收益时，问自己：收益从哪里来？**

---

### 第33页：高 APY 的真相

让我用一个经典例子来说明。

假设你存入 100 美元，协议承诺 100% 年化收益，一年后你获得了价值 100 美元的 Token 奖励。但如果这一年里 Token 价格跌了 80%，你实际收到的只值 20 美元。本金加收益总共 120 美元变成了 20 美元，**你亏了 80%**。

所以遇到高收益项目，你要问三个问题：收益从哪里来？是业务收入还是 Token 增发？Token 会贬值吗？本金安全吗？回答不清楚这些问题，就不要投。

---

### 第34页：4.4 本节要点

这一节的核心内容，让我帮你梳理一下。

区分「总供应量」和「流通量」，关注 FDV 这个指标。大额解锁可能带来巨大抛压，提前关注解锁日历。区分通胀型和通缩型 Token 的不同价值逻辑。最重要的是区分「真实收益」和「Token 排放」，避免被高 APY 迷惑。

记住那个公式：**如果你不知道收益从哪里来，你可能就是那个收益来源**。接下来我们进入最后一节，也是最沉重的一节。

---

## 4.5 为什么大多数 Token 会失败

### 本节讲解思路

这一节要直面一个残酷的事实：**超过 90% 的 Token 最终会失败**。这不是危言耸听，而是统计规律。

我会先告诉你这个数字的含义，然后分析失败的常见原因，重点讲如何识别庞氏结构，最后给你一份实用的「红旗检查清单」。这些内容可能让你不舒服，但了解风险才能保护自己。

---

### 第35页：残酷的事实

我要非常直接地告诉你一个数字：**超过 90% 的 Token 最终会失败**。

这是一个统计事实，不是悲观论调。

---

### 第36页：什么叫「失败」

什么叫「失败」？价格归零或接近归零，项目停止运营，流动性枯竭无法卖出。2017-2018 年 ICO 热潮中诞生的项目，绝大部分已经归零。即使曾经排名前 100 的 Token，也有很多被淘汰了。

**这意味着什么？** 如果你随机购买一个 Token，大概率会亏损。这不是运气不好，而是基本概率。理解这一点，才能做出理性决策。

---

### 第37页：Token 失败的常见原因

Token 失败的原因可以分为几大类。

**无实际用途（空气币）**占了约 40%——没有解决任何真实问题，纯概念炒作。**经济模型不可持续**占了约 25%——高通胀、庞氏结构、激励设计有缺陷。**团队问题**占了约 20%——跑路、能力不足、内部分裂。**技术/安全问题**占了约 10%——合约漏洞被黑。**竞争/监管**占了约 5%——被更好的项目替代或被监管打击。

接下来我挑几个重点展开讲。

---

### 第38页：无实际用途（空气币）

空气币的特征很明显：没有解决任何真实问题，纯概念炒作蹭热点，只有买卖行为没有实际使用场景。

比如蹭「元宇宙」概念的山寨币，蹭名人热度的 Meme 币（大部分），白皮书写得很好看但从不落地的项目。

**判断方法很简单：问自己，如果这个 Token 价格归零，还有人会用这个产品吗？** 如果答案是「不会」，那它大概率是空气币。

---

### 第39页：经济模型不可持续

第二类是经济模型有问题。高通胀无限增发，收益依赖新用户资金（庞氏结构），激励设计有缺陷无法持续。

典型例子就是我们前面讲的 Anchor/UST，承诺「稳定」年化 20% 收益，但收益来源不可持续，最后崩盘。还有那些需要不断拉新才能维持的项目，Token 增发速度远超需求增长。

**判断方法：如果没有新用户进入，收益还能持续吗？** 如果答案是「不能」，那就是庞氏结构。

---

### 第40页：团队问题

团队问题有好几种类型。

**跑路（Rug Pull）**——卷款消失，项目突然停止，团队删除社交媒体跑路。**能力不足**——技术做不出来，运营跟不上，融资后长期没有产品进展。**内部分裂**——核心成员离开，方向出现分歧，创始人因争议被社区抛弃。**动机不纯**——只为圈钱，无心做事。

匿名团队的风险尤其高，虽然中本聪也是匿名的，但对于新项目来说，匿名大大增加了跑路风险。

---

### 第41页：庞氏结构的特征

让我重点讲讲如何识别**庞氏结构**。

庞氏骗局的本质是用后来投资者的钱支付前期投资者回报。在 Crypto 领域，很多项目本质就是这个结构。

**核心特征有四个**：收益来源是新用户资金而非真实业务，必须不断拉新才能维持，承诺不合理的高回报（比如「无风险」年化 50%+），无法清晰解释收益从哪里来。

看到这些特征，就要警惕。

---

### 第42页：快速判断三问

当你遇到一个「高收益」项目时，用这三个问题快速判断。

**第一问：收益从哪里来？** 健康项目会说「交易手续费、借贷利息」，庞氏结构会说「很复杂」或者说不清。

**第二问：如果没有新用户，收益还能持续吗？** 健康项目说「能，只是可能降低」，庞氏结构说「不能，会崩盘」。

**第三问：为什么收益这么高？** 健康项目说「早期阶段加上承担风险」，庞氏结构说「我们的模式很特别」。

问完这三个问题，大多数庞氏结构就现形了。

---

### 第43页：Token 红旗检查清单（1/2）

给你一份实用的红旗检查清单，上半部分。

**团队红旗**：团队完全匿名无法验证身份，团队有负面记录参与过失败或诈骗项目，团队 Token 无锁仓可以随时卖出。

**产品红旗**：没有实际产品只有白皮书和 PPT，解决的是伪问题没有真实需求，路线图一直延期融资后无进展。

有一个红旗不代表一定是骗局，但红旗越多风险越高。

---

### 第44页：Token 红旗检查清单（2/2）

检查清单的下半部分。

**经济红旗**：承诺不合理高收益年化 100%+ 且「稳定」，收益来源不清晰强调拉人返佣，Token 高度集中少数地址持有大量 Token。

**社区红旗**：社区只讨论价格没有产品和技术讨论，批评声音被删除不允许质疑，官方频繁喊单制造 FOMO 情绪。

**我的建议是：如果一个项目有 3 个以上红旗，建议远离**。宁可错过，不要踩雷。

---

### 第45页：建立健康心态

最后，我想帮你建立一个健康的心态。

**失败是常态，成功是例外**。这和创业一样，大多数项目会失败，这是创新的代价。**不要 All-in 任何单一项目**，分散风险永远是正确的策略。**用「亏得起」的钱参与**，投资前问自己：如果这个 Token 归零，我能接受吗？

还要警惕**幸存者偏差**——你听到的都是成功故事，你听不到的是更多人买了归零的币。成功者不代表平均水平。

---

### 第46页：4.5 本节要点

这一节的核心内容，让我最后总结一下。

超过 90% 的 Token 最终会失败，这是基本概率，不是运气问题。失败原因包括无实际用途、经济模型不可持续、团队问题等。学会识别庞氏结构，用「三问法」快速判断。使用红旗检查清单评估项目，超过 3 个红旗就远离。

建立理性预期，用亏得起的钱参与。这不是让你不参与，而是让你明明白白地参与。
