## 5. 内容要点（研发参考）

### 7.1 普通人为什么不该"找好项目"
- 信息不对称问题
- 普通人的认知局限
- 时间成本与机会成本
- 正确的参与心态

### 7.2 项目快速扫描的核心维度
- 团队背景（是否公开、过往项目）
- 融资情况（投资机构、融资轮次）
- 技术可行性（白皮书、代码仓库）
- Token 经济模型（分配、解锁计划）
- 社区活跃度（真实用户 vs 机器人）
- 产品进度（路线图 vs 实际交付）

### 7.3 什么时候可以直接不看一个项目
- 红旗信号清单（Red Flags）
  - 匿名团队 + 高收益承诺
  - 代码未开源且无审计
  - Token 高度集中
  - 社区充斥机器人
  - 白皮书空洞无实质
  - 已有负面新闻/黑历史

### 7.4 信息源与噪音的区分方法
- 一级信息源：官方 Twitter、Discord、Telegram
- 二级信息源：媒体、KOL、研报
- 数据平台：RootData、DefiLlama、Dune、Token Terminal
- 如何识别付费软文和虚假信息
- 建立个人信息筛选系统
